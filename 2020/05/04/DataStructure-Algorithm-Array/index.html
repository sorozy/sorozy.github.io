Array介绍

  • 数组（Array）是一种线性表数据结构。它用一组连续的内存空间，来存储一组具有相同类型的数据
  • 数组存储的内存模型上来看，“下标”最确切的定义应该是“偏移（offset）”
  • 连续的内存空间和相同类型的数据

根据下标随机访问的时间复杂度为 O(1)

要想在数组中删除、插入一个数据，为了保证连续性，就需要做大量的数据搬移工作

  • 低效的“插入”和“删除”

O(n)

插入改进：直接将第 k 位的数据搬移到数组元素的最后，把新的元素直接放入第 k 个位置

删除改进：记录数据已经被删除。当数组没有更多空间存储数据时，我们再触发执行一次真正的删除操作，这样就大大减少了删除操作导致的数据搬移

与ArrayList比较

（1）最大的优势就是可以将很多数组操作的细节封装起来，支持动态扩容

（2）完全不需要关心底层的扩容逻辑

（3）扩容操作涉及内存申请和数据搬移，比较耗时，最好在创建 ArrayList 的时候事先指定数据大小

（4）Java ArrayList 无法存储基本类型

为何数组从0开始编号

“下标”最确切的定义应该是“偏移（offset）”

a[0]就是偏移为 0 的位置，也就是首地址

原因：减少一次减法指令

线性表与非线性表

  • 线性表（Linear List）

数据排成像一条线一样的结构。每个线性表上的数据最多只有前和后两个方向

线性表数据结构：数组，链表、队列、栈等

  • 非线性表

二叉树、堆、图等

在非线性表中，数据之间并不是简单的前后关系
